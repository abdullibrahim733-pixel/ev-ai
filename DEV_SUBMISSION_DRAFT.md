## What I Built

**EV-AI** is an autonomous electric vehicle 3D driving simulator — entirely hand-built from scratch. An AI-controlled EV navigates a procedurally generated track using sensor raycasting, pure pursuit path following, and obstacle avoidance. The 3D model itself is generated by Python code (GLTF Binary format), not downloaded from any asset store.

Every line of code — from the 3D mesh vertices of the car body to the AI agent's state machine — was written by me. This is a showcase of what happens when passion meets a weekend and a willingness to build everything from first principles.

**Live demo:** [https://ev-ai.vercel.app](https://ev-ai.vercel.app)

**Tech stack:**
- **3D Engine:** p5.js WEBGL
- **3D Model:** Python custom GLTF/OBJ generator (raw vertex construction — no Blender, no CAD)
- **AI Agent:** Custom sensor raycasting (8 directional), pure pursuit path following, state machine
- **AI Co-pilot:** Google Gemini API for natural language conversations with the vehicle
- **UI:** CAD-grade interface — SVG gauges, neural net visualization, orientation cube
- **Backend:** Python serverless on Vercel (model serving + simulation API + Gemini proxy)
- **Deployment:** Vercel with GitHub auto-deploy

### The AI Agent System

| Component | Description |
|-----------|-------------|
| **Sensors** | 8 directional raycast sensors (front, sides, rear) for environment perception |
| **Path Following** | Pure pursuit algorithm with 64 waypoints |

## How I Built It

### The 3D Model — Raw Python Generation

Instead of downloading a car model from an asset store, I wrote a Python script that constructs the 3D mesh vertex by vertex. The model includes:
- Car body with aerodynamic curves
- Cabin with windshield
- Four wheels with visible rims
- Headlights and taillights
- Side mirrors

The generator outputs both GLTF Binary (`.glb`, 19.5 KB) and Wavefront OBJ (`.obj`, 43 KB) formats by writing the binary specifications directly — no 3D library dependencies.

**Why?** Because passion is about understanding your tools from the ground up. Using someone else's model would have been faster, but building my own taught me the geometry of GLTF, the structure of OBJ, and how 3D rendering actually works under the hood.

### The AI Agent

The autonomous driving system uses:

1. **8 Raycast Sensors** — cast in a fan pattern around the car (front, front-left, front-right, sides, rear-left, rear-right). Each sensor returns distance to the nearest obstacle or path boundary.

2. **Pure Pursuit Path Following** — 64 waypoints define a closed track. The agent selects a lookahead waypoint based on current speed and curvature, then calculates the steering angle needed to reach it.

3. **State Machine** — three states:
   - `FOLLOW` — normal path following
   - `AVOID` — obstacle detected, steer around
   - `RECOVER` — realign after avoidance maneuver

4. **Adaptive Speed Control** — slows down on sharp curves, accelerates on straights. Eco mode activates below 15% battery to extend range.

### Google AI Co-pilot (Best Use of Google AI)

This is where the project gets really interesting. I integrated **Google's Gemini API** to create a conversational AI Co-pilot inside the simulation.

**How it works:**

When you type a question (e.g., "How's my battery?" or "What's the car thinking right now?"), the frontend collects the full simulation telemetry — speed, battery, AI state, decision, waypoint progress, and all 8 sensor readings — and sends it to a serverless Python endpoint that proxies the request to the Gemini API.

Gemini receives:
- A **system prompt** that defines it as an in-vehicle AI assistant for an autonomous EV
- The **live telemetry** as structured context
- The **user's question**

It then responds in natural language — grounded in the actual state of the vehicle.

**Example interactions:**

| You Say | Co-pilot Responds |
|---------|------------------|
| *"How's my battery?"* | "85% remaining — about 42 km range. You're well within eco range." |
| *"What's the car doing?"* | "Following the path at 65 km/h. Waypoint 23/64 ahead. AI confidence is 94%." |
| *"Tell the car to speed up"* | "Throttle increased to 0.7. Target speed is now 80 km/h." |
| *"Explain pure pursuit"* | "Pure pursuit calculates the steering angle needed to reach a lookahead point on the path ahead — like chasing a moving target." |

**Why I chose Google AI:**
- Gemini's API is straightforward to integrate — a single POST request with system instructions
- It handles structured context well, understanding both telemetry data and natural language
- The responses are fast enough for real-time chat, even through a serverless proxy

**Architecture:**
```
User types question → Frontend collects telemetry → POST /api/ai/chat
→ Python serverless function → Gemini API (gemini-2.0-flash)
→ Response displayed in chat panel
```

This project is the result of two things I'm deeply passionate about:

3. **AI that actually understands context** — The Gemini co-pilot isn't a gimmick. By feeding it live telemetry, it gives genuinely useful, grounded answers about the vehicle's state. That's the kind of AI I want to build more of.

I'm a 19-year-old mechatronics engineering student from Tanzania, and I believe the next generation of transformative technology won't just come from Silicon Valley — it will come from anyone, anywhere, with enough passion to build it.
